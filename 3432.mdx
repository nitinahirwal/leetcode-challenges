- Left: `nums[0â€¦i]`

## ğŸ’¡ Intuition  

Let:

- `L = sum(left)`
- `R = sum(right)`
- `S = total sum = L + R`

The difference is:

\[
\text{diff} = L - R = L - (S - L) = 2L - S
\]

Hereâ€™s the key observation:

### âœ… `2L` is always even  
So the parity of `diff` depends **only on `S`**.

Therefore:

- If **total sum `S` is even â†’ diff is always even â†’ all `(n - 1)` partitions are valid**
- If **total sum `S` is odd â†’ diff is always odd â†’ no partition works**

This turns the problem into a simple sum and parity check!

---

## ğŸ”‘ Approach  

1. Compute the total sum `S` of the array.
2. If `S` is **even**, return `n - 1` â€” because every split works.
3. If `S` is **odd**, return `0` â€” because no split can produce an even difference.
4. Done.

No prefix sums, no checking each split, nothing complicated.

var countPartitions = function(nums) {
    const n = nums.length;
    let total = 0;
    for (let x of nums) total += x;
    return (total % 2 === 0) ? (n - 1) : 0;
};
